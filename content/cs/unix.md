## 1. 유닉스의 특징

- 대화형 시스템
- 다중사용자 시스템
- 다중 작업용 시스템
- 높은 이식성과 확장성 제공
- 계층적 트리 파일 시스템

## 2. 유닉스의 구성요소

- 커널 : 프로세스 관리, 메모리 관리, 파일 시스템 관리, 장치 관리 등 컴퓨터의 모든 자원을 초기화하고 제어하는 기능을 수행 함.
- 셸 : 사용자와 커널 사이의 중간자 역할을 담당하는 프로그램으로 사용자가 입력한 명령을 해석하여 커널에 넘겨줌.
    - 본 셸(bourne shell), C 셸, 콘 셸(korn shell), 배시 셸(bash shell)등 다양함

## 3. 유닉스의 파일 시스템

- 유닉스는 디렉터리 파일, 일반 파일, 장치 파일이 있습니다.
- 이 모든 파일은 하나의 트리 구조를 가지며 계층적으로 관리합니다.

## 4. 디스크 블록 구조

- 부트 블록 : 파일 시스템에 유닉스 커널을 적재시키는 프로그램을 포함합니다.
- 슈퍼 블록 : 파일 시스템을 관리하는 정보를 저장합니다.
- I-노드 블록 : 파일 관리에 필요한 정보를 저장합니다.
- 데이터 블록 : 일반 파일이나 디렉터리 파일의 내용이 들어있습니다.

## 5. 유닉스 명령어

- cat : 텍스트 파일 내용을 출력합니다. 
- cd : 디렉터리를 이동합니다.
- cfgadm : 유닉스 기반의 관리를 설정합니다.
- chgrp : 파일이나 디렉터리의 그룹을 변경합니다.
- chmod : 파일의 접근 권한을 변경합니다.
- chown : 파일 사용자와 그룹을 변경합니다.
- clear : 터미널의 텍스트 화면을 깨끗이 지웁니다.
- cmp : 파일을 비교하여 다른 부분을 알려줍니다.
- compress : 파일을 압축합니다.
- cp : 파일을 지정된 경로에 복사합니다.
- df : 파일시스템 디스크 공간의 사용량을 출력합니다.
- diff : 파일을 줄 단위로 비교하여 출력합니다.
- dig : 네임 서버(DNS) 진단을 위한 룩업 유틸리티입니다.
- finger : 사용자 정보를 출력합니다.
- fsck : 파일시스템을 점검하고 복구합니다.
- grep : 특정 문자열을 검색하여 동일한 문자열이 있는 줄의 패턴을 출력합니다.
- init : 부팅할 때 실행 레벨에 따라 프로세스를 호출합니다.
- kill : 시스템에 예기치 않은 문제가 생긴 프로세스를 종료시킵니다.
- ln : 파일 링크를 만듭니다.
- ls : 디렉터리 목록을 출력합니다.
- man : 알고싶은 명령어의 매뉴얼을 출력합니다.
- mkdir : 디렉터리를 생성합니다.
- mkfs : 리눅스 파일 시스템을 만듭니다.
- more : 파일을 화면 단위로 출력합니다.
- mount : 디바이스와 파일 시스템을 연결합니다.
- mv : 파일이나 디렉터리를 이동하거나 이름을 변경합니다.
- netstat : 네트워크의 연결과 포트를 출력합니다.
- nice : 프로세스의 스케줄링 우선순위를 변경합니다.
- nslookup : 네임 서버 설정이 제대로 동작하는지 확인합니다.
- passwd : 사용자의 패스워드를 변경합니다.
- ping : ICMP_ECHO_REQUEST 메시지를 네트워크 호스트로 보냅니다.
- ps : 프로세스의 현재 상태를 출력합니다.
- pstree : 프로세스의 상관 관계를 트리 형태로 출력합니다.
- pwd : 현재 작업 중인 디렉터리의 절대 경로를 출력합니다.
- rm : 파일과 디렉터리를 삭제합니다.
- rmdir : 빈 디렉터리를 삭제합니다.
- sftp : 보안 암호화하여 파일을 전송합니다.
- shutdown : 시스템을 안전하게 종료합니다.
- strace : 시스템 콜을 추적하여 프로그램의 실행 과정을 출력합니다.
- su : 로그아웃 없이 임시로 다른 사용자의 UID, GID 환경을 사용합니다.
- sudo : 다른 사용자로 명령을 실행합니다.
- tar : 여러 파일을 묶거나 해제합니다.
- touch : 빈 파일을 생성하거나 기존 파일의 시간을 변경합니다.
- unmount : 마운트한 장치를 해제합니다.
- uname : 시스템의 정보를 출력합니다.
- useradd : 시스템에 사용자를 추가합니다.
- vmstat : 가상 메모리의 정보를 통계 형식으로 출력합니다.
- whereis : 명령어를 입력받아 해당 명령어의 절대 경로를 찾아줍니다.
